  What was happening:

  1. The cycling bug: The medic was stuck in a loop cycling between stand and run states when it had no enemy
  2. Root cause: In g_ai.cpp, when ai_stand's pausetime expires, it checks if the monster is summoned in horde mode and
  calls run() instead of walk()
  3. The problem: Summoned medics would go to run state even with no enemy, then medic_run would be called repeatedly,
  incrementing pausetime each frame

  What fixed it:

  The fix catches when medic_run is called with no enemy and no healing target, and:
  1. Sets a 1-second pausetime to stabilize the AI
  2. Switches back to stand animation
  3. Returns early to prevent the rest of the run logic

  This breaks the cycle and gives the AI time to properly find targets through the normal ai_stand routines instead of
  getting stuck in the stand->run->stand loop.

  The other fixes we applied earlier (removing forced pausetime in medic_idle and medic_finish_and_hunt) helped with the
  healing/resurrection flow but this specific fix addressed the core cycling issue.
